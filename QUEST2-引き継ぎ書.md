# 直幸QUEST2 引き継ぎ書（Fable 5 → Sonnet 4.6）

## このファイルの目的
シーズン2試作版の土台はFable 5で作成済み。
ここから先の細かい開発はSonnet 4.6が担当する。
新しいセッションでこのファイルを読めば、経緯を知らなくても作業を続けられる。

## 対象ユーザー
- 11歳男児（直幸くん）
- 本編「直幸QUEST」（naoyuki-quest-ipad.html）をクリアした後に遊ぶ続編
- 目的：日々の生活習慣（クエスト）への意欲づけ

## シーズン2のコンセプト
- タイトル：『直幸QUEST2 〜南の新大陸と光の絆〜』
- 本編第8章ラスト（ナオとライが船で新大陸へ出発）から直結
- 主役は **ナオとライの二人**。元ヤミノ王のライが「百年サボった者」として
  毎日の積み重ねの尊さを学び直す、友情と再起の物語
- 感動的なストーリー ＋ 白熱の3Dバトルが二本柱

## 試作版の場所と構成
ファイル：`naoyuki-quest2-prototype.html`（1ファイル完結、本編と同じ方式）

実装済みの機能：
1. **3D背景**（Three.js r149 / CDN読み込み）
   - 夜の海に浮かぶ新大陸の島（ローポリ）、揺れる波、星空
2. **ホームタブ**：クエスト9個（本編と同じタスク・配点）、累積/今日/連続日数
3. **絆ゲージ**：今日のクエスト達成率＝ナオとライの絆。
   100%でバトルの合体技「ダブル・ライト」が解放
4. **2D ATBバトル（FF6風）**：vs シャドウ・ウルフ（本編の画像を流用）
   ※3Dバトルは「棒人間にしか見えない」とのオーナー判断で撤去。
     2D路線が確定。コードで人型3Dを作る提案は今後しないこと。
   - ATBゲージ：速さに応じて溜まり、満タンでコマンド選択（ウェイト式＝
     メニューを開いている間は時間停止。子ども向けに考えて選べる）
   - コマンド：たたかう／とくぎ・まほう／アイテム／ぼうぎょ（半減）
   - レベル＝累積ポイントから自動計算 heroLv()=1+floor(sqrt(total/25))
   - 技・魔法はSKILLS定数で管理、unlockPt（累積ポイント）に達すると習得。
     未習得は「？？？ ◯Pで習得」と表示され先の楽しみになる
   - ダブル・ライト：絆ゲージ100%の日だけ使える合体技（1戦闘1回）
   - アイテム：ITEM_DEFSのevery値ごとに自動支給（100P毎ポーション、
     300P毎エーテル、1000P毎エリクサー）。state.inv / state.granted で管理
   - クリティカル率＝連続日数×2%（最大25%）。生活習慣が強さに直結する設計
   - 演出：敵明滅・画面シェイク・ダメージ数字ポップ・技名カットイン・
     突進アニメ（CSS）。敵画像の白背景は mix-blend-mode:multiply で消し、
     背後に #foeGlow（月光）を置いて視認性を確保
   - 敵HPバーは上段に表示。倒すとWIN演出＋40%でポーションドロップ
   - 仲間モンスター枠：4人PTのうち2枠は「なかま募集中」表示。
     画像が用意でき次第 B.heroes[2],[3] に追加する設計
   - **リミット技**：HP30%以下のピンチで1戦闘1回だけ使える（LIMITS定数）。
     ナオ＝クライシスブレード（atk×6の大ダメージ）、
     ライ＝ひかりのいのり（全員60%回復＋光攻撃）。
     ピンチ時はコマンド欄にオレンジに光るボタンが出現
   - **節ごとの敵出現**：ENCOUNTERS定数。累積ポイントで敵が切り替わり、
     敵画像・戦闘背景（章の画像）・強さ倍率(mul)もセットで変わる。
     QUEST2用の敵画像が無いため「ライの幻影魔法で過去の強敵と修行」
     という設定で本編のボス画像を流用中。画像が揃ったらここを差し替える
   - **サウンド（Web Audio・ファイル不要）**：
     チップチューンBGM2曲（battle=160bpm緊迫曲 / home=84bpm穏やか曲）を
     playBgm()でループ再生。SFX定数に効果音10種（攻撃・被弾・回復・
     ターン開始・カットイン・リミット・アイテム・メニュー・勝利・敗北）。
     iPadの制約で初回タップ後に音が出る（pointerdownでaInit）。
     右上に🔊/🔇ミュートボタン（state.mutedで永続化）
5. **ストーリータブ**：第1章「二人の船出」全5シーン（ポイント解放式）

## 技術メモ（重要）
- localStorage キー：`naoyuki_quest2_proto`（本編の `naoyuki_battle3` とは別。本編データを壊さない）
- Three.js は **r149のthree.min.js** を使用。r150以降はこの形式が廃止されているので
  バージョンを上げる場合は ES Modules（importmap）方式への書き換えが必要
- 画像ファイルを追加する場合、**GitHub Pagesは大文字小文字を区別する**。
  実ファイルの拡張子（.PNG / .png）とHTML内の参照を必ず一致させること
  （本編で何度もこれが原因の表示バグが起きた）
- macOSは大文字小文字を区別しないので、git addで混乱しやすい。注意
- ホスティング：GitHub Pages（リポジトリ 724minimal-gif/naoyuki-quest）
- iPadのSafariブックマークで起動する運用（PWA・Service Workerは使わない。
  本編でキャッシュ問題が起きたため撤去した経緯あり）
